# 第二十部分C-1｜各大社交媒体平台内容创作技巧与规则

## PC-0. 使用边界：规则与技巧不是一回事

- **规则**来自法律法规和平台当期公示，必须遵守并动态核验。
- **技巧**是锦城豪宅的内部测试建议，不是平台官方算法承诺。
- 本节出现的时长、字数、页数和频率均为**建议测试区间**，应以账号后台数据迭代；不得对外宣称“平台规定必须如此”。
- 每次大规模投放、直播、商业合作、联系方式承接或账号认证前，重新检查平台规则中心。抖音已单列房产内容管理规范；小红书、微信、哔哩哔哩、知乎、微博、快手和今日头条也分别有社区或创作者规则。[^platform-douyin][^platform-xhs][^platform-wechat][^platform-bilibili][^platform-zhihu][^platform-weibo][^platform-kuaishou][^platform-toutiao]

## PC-1. 全平台内容宪法

### 事实与判断

1. 把内容拆成 `facts / inferences / unknowns`。
2. 房源价格、在售状态、交付、面积、景观、学区/配套、规划和成交记录必须标日期。
3. 不使用“稳赚、必涨、抄底、最后机会、全成都第一、永久稀缺”等无法证明或可能误导的表达。
4. 房地产广告不得承诺升值或投资回报；规划中配套、效果图、样板和未实现物业服务应清楚说明。[^compliance-realestate]
5. 商业推广应让用户能够识别其商业性质；合作、代理、广告和客户授权边界需留档。[^compliance-ad]

### 隐私与版权

1. 未获授权不得展示客户、业主、邻居、家庭成员的人脸、声音、姓名、车牌、门牌、证件、聊天、账户、行踪和家庭关系。
2. 豪宅内部的安防、贵重藏品位置、长期空置规律和可推断住址的信息默认不公开。
3. 处理金融账户、行踪轨迹等敏感个人信息，应遵循特定目的、充分必要和严格保护；内容生产只收集最小必要信息。[^compliance-privacy]
4. 音乐、图片、户型图、设计图、航拍、媒体报道和他人视频必须有可证明的授权或合法使用依据。
5. 不用“同行素材改字”“搬运剪辑”“批量洗稿”代替原创。

### 线索与互动

1. 优先使用平台内私信、预约、专业号、企业号、小程序等官方能力。
2. 不通过谐音、拆字、背景二维码、头像水印、评论暗号等方式规避平台导流规则。
3. 不买粉、刷赞、刷评论、虚构咨询、虚构成交或诱导互动。
4. 评论区不讨论客户身份、精确房号、不可公开底价和未核验纠纷。
5. 对外回应争议只讲事实、证据、边界和处理进展，不与用户或同行情绪对骂。

## PC-2. 标准内容生产流水线

```text
客户真实问题/市场变化
→ 选题立项与内容 ID
→ 证据包（数据、房源、案例、授权）
→ 平台原生 Brief
→ 拍摄/设计前隐私与版权检查
→ 初稿：事实/推断/未知分层
→ 专业审稿 + 品牌审稿 + 必要时合规审稿
→ 发布与首轮评论管理
→ 24/72 小时数据复盘
→ 线索进入 CRM
→ 7/30 天长尾与事实更新
```

每条内容至少保存：原素材、授权、数据来源、最终稿、发布时间、平台链接、客户咨询、修改记录和下次复核日期。

## PC-3. 选平台的决策逻辑

| 客户任务 | 首选平台 | 辅助平台 |
|---|---|---|
| 让更多成都/四川高净值客户知道马老师 | 抖音、视频号、快手 | 微博、今日头条 |
| 影响配偶和家庭生活决策 | 小红书、视频号 | 公众号、朋友圈 |
| 建立复杂交易和资产判断权威 | 公众号、哔哩哔哩、知乎 | 视频号、今日头条 |
| 获得业主委托 | 视频号、朋友圈、抖音 | 公众号、小红书 |
| 建立成都南门搜索资产 | 小红书、知乎、公众号、哔哩哔哩 | 抖音、今日头条 |
| 发布时效市场判断 | 抖音、微博、朋友圈 | 视频号、公众号 |
| 维护老客和转介绍 | 视频号、朋友圈、企业微信 | 公众号、小程序 |

原则：**一个选题只有一个首要客户任务；平台越多，越不能一稿原样复制。**

## PC-4. 抖音：结论前置、现场证据与本地搜索

### 内容结构

建议测试三类长度：

- **25—60 秒**：一个反常识结论或一个房源判断。
- **60—120 秒**：三点比较、项目优缺点、买/不买边界。
- **90—180 秒**：脱敏案例或复杂交易的完整转折。

短视频脚本：

```text
0—3 秒：预算/板块/冲突 + 明确结论
3—15 秒：为什么大多数人判断错
15—60 秒：2—3 个事实证据或现场验证
60—90 秒：适合谁 / 不适合谁
结尾：低压力 CTA + 信息日期
```

### 创作技巧

- 竖屏优先，人物近景与空间证据交替；不要从 15 秒空镜开始。
- 口播、字幕、标题和画面中使用一致的搜索词，例如“成都豪宅、麓湖、金融城、3000 万、大平层”。
- 每 5—8 秒给一个信息变化：镜头、图纸、数据、现场缺点或客户问题。
- 封面只承载一个冲突，例如“3000 万，为什么不选返佣最高的？”
- 项目视频至少讲一个不可修复缺点，避免变成售楼部宣传片。

### 直播规则与技巧

- 直播前核验房源状态、价格、拍摄授权和可公开区域。
- 直播中不展示门牌、住户、人脸、车牌、安防和非公开房号。
- 不口头承诺底价、升值、学区、规划、交付或贷款结果。
- 由场控维护“已核验 / 待核验 / 不能公开”问题清单。
- 直播结束把高频问题沉淀为 FAQ 和下一周短视频。

### 转化与 KPI

CTA 以平台内私信、预约诊断和官方组件为主，不做隐蔽外链。核心指标：有效私信、5 秒留存、完播、收藏/分享、直播有效咨询、诊断率和内容辅助成交，而不是只看播放量。抖音规则中心包含房产内容管理规范，发布前应复核当期要求。[^platform-douyin]

## PC-5. 小红书：搜索问题、家庭场景与可收藏清单

### 推荐形式

- 图文：建议测试 **6—12 页**，第一页给结论，后续每页只解决一个决策点。
- 视频：建议测试 **45—120 秒**，强调生活动线、审美、收纳、家庭成员和真实缺点。
- 标题：城市/板块/预算/家庭问题四项中至少包含两项，不堆叠情绪符号。

### 图文模板

```text
封面：一个真实问题
第 2 页：一句话结论
第 3—5 页：适合谁、产品和生活场景
第 6—8 页：价格/交付/物业/流动性风险
第 9—10 页：替代选项
末页：核验日期 + 低压力咨询入口
```

### 创作技巧

- 把“豪”翻译成家庭感受：孩子、长辈、家政、收纳、车库、宠物、宴客、隐私和日常维护。
- 每篇至少提供一个可保存的表格、清单或判断框架。
- 使用自然搜索词，不把标题写成硬广口号。
- 实景图、效果图、样板间和示意图必须清楚区分。
- “亲测、真实入住、业主视角”等第一人称表述必须与实际身份一致。

### 规则与转化

小红书社区规范不鼓励营销和导流信息，并对交易导流、侵权、不当行为等设有约束；优先使用专业号和平台内线索工具，不在图片、评论或个人资料中隐藏联系方式。[^platform-xhs] 核心指标：搜索曝光、收藏率、30 天长尾咨询、家庭决策者占比和收藏到诊断转化。

## PC-6. 微信视频号：可转发给家人的可信解释

### 内容结构

建议测试 **60—180 秒**，比抖音更完整、更克制：

```text
客户遇到什么问题
→ 马老师先做了什么判断
→ 哪个证据改变了决策
→ 最后如何处理
→ 这套方法适合/不适合谁
```

### 创作技巧

- 开头用“这位客户真正纠结的不是价格，而是……”代替强刺激标题。
- 让内容可被客户直接转发给配偶、父母或合伙人，不使用过多行业黑话。
- 重要观点同步到朋友圈和公众号形成证据闭环。
- 直播适合周度市场问答、项目专题和老客私享；直播预约、转发和回放都应低压力。

### 规则与 KPI

遵守微信公众平台和视频号相关运营规范，不诱导分享、关注或用虚假信息获取传播。[^platform-wechat] 重点看私域完播、转发、家庭二次观看、转介绍、预约和老客复购。

## PC-7. 微信公众号：可引用、可更新的权威档案

### 推荐结构

普通深度文章建议测试 **1500—3500 字**；项目/市场报告可更长，但必须有目录和摘要。

```text
标题：城市/板块/预算 + 问题
摘要：三句话结论
01 事实与数据日期
02 客户场景
03 产品/项目比较
04 风险与反例
05 马老师建议与不建议
附录：数据来源、更新日、咨询入口
```

### 创作技巧

- 用表格、决策树和项目卡，减少营销形容词。
- 旧文章每季度检查房源、价格、规划、政策和交付事实；过期内容标注更新。
- 重要报告区分公开版与客户版；客户版可增加个性化结论但不得群发敏感数据。
- 文章与小程序、视频号和 CRM 使用同一 `content_id`。

### 规则与 KPI

禁止诱导分享/关注、欺骗性链接、虚假信息和不清晰的商业表达；合作内容应保留审稿和授权记录。[^platform-wechat] 核心指标：阅读完成、收藏、转发、搜索进入、文章辅助诊断和成交。

## PC-8. 朋友圈与企业微信：工作证据，不是广告墙

### 建议内容配比

每 7 条可测试：

- 3 条专业判断：板块、房源、交易、叫停。
- 2 条工作过程：核验、谈判、交付、复盘。
- 1 条客户价值：脱敏结果或客户评价。
- 1 条个人文化：书法、阅读、城市生活、团队。

### 规则

- 不连续刷同一房源，不群发无差别“急售”。
- 不发客户当天行程、精确总价、房号、聊天截图和家庭矛盾。
- 对不同标签客户分层可见；高敏业主房源只在受控名单内传播。
- 一对一发送资料前先说明为什么与对方有关；不把企业微信变成自动轰炸工具。
- 朋友圈线索同样进入 CRM，记录触发内容和下一步。

核心指标：有效回复、主动询问、约谈、业主委托、转介绍和长期客户唤醒。

## PC-9. 哔哩哔哩：深度、章节与长期搜索

### 推荐形式

建议测试 **6—15 分钟**，重要项目可做系列。视频必须有章节：

```text
结论 → 板块 → 产品/户型 → 实景验证 → 价格与替代
→ 风险 → 适合/不适合 → 资料日期
```

### 创作技巧

- 标题解决一个明确问题，不写纯形容词：“麓湖 370㎡次顶，3000 万买到的到底是什么？”
- 封面一张主视觉 + 6—10 个核心字，避免堆太多数字。
- 户型、动线、剖面、景观和交通用图解；来源与示意性质标清。
- 内容中保留缺点、反例和待核验项，建立专业评论区。
- 评论区高质量问题进入下一期，不用复制粘贴营销回复。

### 规则与 KPI

遵守哔哩哔哩社区规则、用户协议、原创和版权要求；商业推广和素材授权需可追溯。[^platform-bilibili] 核心指标：平均观看时长、章节留存、搜索播放、高质量评论、收藏和跨平台咨询。

## PC-10. 知乎：先回答问题，再建立专家身份

### 写作结构

建议测试 **1200—3000 字**，复杂问题可更长：

```text
先给结论
→ 说明适用条件
→ 用框架拆解
→ 给一至两个脱敏案例
→ 讲反例与风险
→ 列待核验信息
→ 低压力 CTA
```

### 创作技巧

- 选择已有真实问题，不自问自答制造虚假讨论。
- 标题和正文使用用户搜索语言：“成都预算 3000 万，麓湖和金融城怎么选？”
- 不把多个问题下复制同一回答；根据提问重写角度。
- 引用数据注明来源和日期；区分经纪人经验与事实。
- 机构号内容保持专业，不通过高频私信、评论和链接骚扰导流。

知乎社区鼓励真诚、专业、友善讨论，反对低质、抄袭、侵权、虚假认证和恶意营销导流。[^platform-zhihu] 核心指标：长尾搜索、收藏、专业引用、站内有效私信和内容辅助决策。

## PC-11. 微博：时效观点、市场快讯和品牌公关

### 推荐结构

- 单条：**一个事实 + 一个判断 + 一个边界**。
- 复杂话题用线程/长文，不在一条中堆满结论。
- 图片建议使用数据卡、项目现场或活动图，不用未经授权的聊天和客户照片。

### 创作技巧

- 适合发布当天市场变化、项目节点、公开活动、媒体观点和快速纠错。
- 参与热搜必须与成都豪宅业务真实相关，不借无关社会事件硬蹭。
- 未核验爆料、纠纷和截图不发布；涉及同行和项目只陈述可证明事实。
- 发生信息错误时，原帖更正并保留时间线，不悄悄删除造成二次风险。

微博社区公约重点处理不实信息、侵害权益、诈骗、违规营销、作弊和垃圾信息等。[^platform-weibo] 核心指标：品牌搜索、专业转发、媒体引用、圈层触达和危机响应速度。

## PC-12. 快手：区域真实感、持续直播与四川客源

### 适用场景

重点测试四川地级市、县域企业家、返蓉和区域改善客群；表达可以更生活化，但不能刻意模仿或贬低地域形象。

### 创作技巧

- 30—120 秒真实口播、现场走访和客户问题复盘。
- 用“老板从绵阳/宜宾到成都，真正需要解决什么”讲跨城场景。
- 直播强调稳定固定栏目、真实回答和复访观众，而非单场喊价。
- 封面简洁，人物和主题明确；不放二维码、电话号码和隐蔽联系方式。

快手社区规则明确禁止侵害隐私、未经授权搬运和强行跟拍等行为。[^platform-kuaishou] 核心指标：四川区域有效线索、直播咨询、复访观众、转介绍和到店/诊断率。

## PC-13. 今日头条：长图文、搜索推荐与中长内容分发

### 推荐形式

- 图文：1000—2500 字市场解释、板块科普、案例和清单。
- 微头条：一个时效事实 + 一句话判断 + 来源/日期。
- 视频：2—8 分钟的项目与交易知识，可由长视频重新剪辑但需原生标题和开头。

### 创作技巧

- 标题直接回答用户问题，不用惊悚、悬念和夸张数字。
- 保持领域垂直，城市、板块、预算和客户场景清楚。
- 文章开头 100 字给核心结论；中段使用小标题、表格和案例。
- 不把未经核验的行业传闻写成“新闻”；重要市场事实引用官方来源。
- 账号简介和正文不以违规方式放联系方式，使用平台允许的承接方式。

今日头条社区规范要求遵守法律政策，反对虚假、低质、侵权和不当营销内容。[^platform-toutiao] 核心指标：阅读完成、搜索流量、长尾咨询、账号垂直度和跨平台品牌搜索。

## PC-14. 全平台直播 SOP

### 开播前

1. 确认主题、受众和唯一 CTA。
2. 核验所有房源、价格、项目和数据的日期。
3. 获得场地、业主、开发商及人物拍摄授权。
4. 准备“可答、待核验、不可公开”三类问题表。
5. 设主持、主讲、场控、评论审核和线索记录责任人。
6. 测试网络、收音、画面、灯光、备用电源和应急停播。

### 建议 60—90 分钟流程

```text
0—5 分钟：定位与今天能解决的问题
5—20 分钟：市场/板块事实
20—45 分钟：三个案例或项目比较
45—70 分钟：高质量问答
70—80 分钟：总结适合/不适合与下一步
```

### 直播红线

- 不公开非授权住户、门牌、安防、证件、合同、银行或客户行程。
- 不制造“直播间独家底价”而无书面依据。
- 不承诺收益、政策结果、贷款、交付和学区。
- 不回答未核验的具体纠纷；说明核验后再回复。
- 出现隐私、违法、重大事实错误或场地安全问题立即切画面/停播。

### 直播后

24 小时内：整理问题、有效线索、待核验事项、时间戳切片、违规风险和下次内容；每条线索进入 CRM，不通过评论区公开追问个人资产信息。

## PC-15. 一题多端原生改写规则

以“3000 万以内，大平层还是整层”为例：

| 平台 | 原生表达 |
|---|---|
| 抖音 | 60—90 秒，先给结论和三种冲突，现场/图纸作证 |
| 小红书 | 8—12 页，按家庭人数、空间、交付、物业和维护做清单 |
| 视频号 | 2—3 分钟脱敏案例，便于转发给配偶和家庭成员 |
| 公众号 | 深度比较，包含数据、合同/交付风险和决策表 |
| 朋友圈 | 一张结论卡 + 一段当日工作判断，不贴整篇硬广 |
| 哔哩哔哩 | 8—15 分钟户型、动线、实景与价格替代完整比较 |
| 知乎 | 回答高意图问题，先讲适用条件和反例 |
| 微博 | 一条事实速评 + 线程补充，不做长篇复制 |
| 快手 | 真实口播，结合四川企业家家庭的具体使用场景 |
| 今日头条 | 1200—2000 字图文或 3—6 分钟知识视频 |
| 私域客户版 | 一页客户专属对比，只保留与其需求相关的结论 |

禁止只改标题、正文和画面完全相同的“机械一稿多发”。

## PC-16. 标题、封面和开场的统一规则

### 标题公式

```text
对象/预算/板块 + 真实冲突 + 判断边界
```

好标题：

- “3000 万成都改善，为什么整层不一定比大平层更适合？”
- “麓湖别墅看了一年半，最后解决的不是房源数量。”
- “豪宅业主急卖时，什么时候不应该立刻降价？”

禁止：

- 无证据“全网最低、成都第一、最后一套、闭眼买”。
- 用客户隐私、冲突八卦或财富炫耀换点击。
- 标题承诺正文没有的信息。

### 开场六种方式

1. 结论：“这套房产品很好，但我不建议三类人买。”
2. 冲突：“客户预算 3000 万，最大的竞争对手不是另一个项目，而是返佣。”
3. 反例：“豪宅挂牌越多，不代表越容易买到好资产。”
4. 时间点：“今天这条只讲截至某日已核验的三个变化。”
5. 问题：“一家五口买大平层，最容易漏掉哪条动线？”
6. 案例：“这位客户看了一年半，最后我们没有继续找房。”

## PC-17. 内容 SEO 与可检索资产

每条内容选择 1 个主关键词 + 2—4 个辅助词：

```text
主词：成都豪宅 / 麓湖 / 金融城 / 天西 / 项目名
辅助：预算、别墅、大平层、二手、交付、业主委托、企业家家庭
问题词：怎么买、值不值、适合谁、怎么卖、风险、对比
```

主词应自然出现在：标题、口播前段、字幕/正文、封面或首图、标签和文件名中；不同平台不堆砌无关热门词。每月检查“马老师、马国良、锦城豪宅、核心板块、项目名”的搜索结果一致性。

## PC-18. 评论、私信与线索承接

### 评论区分层

- A 类：具体预算、板块、项目、时间 → 邀请平台内私信诊断。
- B 类：专业问题 → 公开回答，沉淀 FAQ。
- C 类：争议/质疑 → 给证据和边界，必要时更正。
- D 类：隐私、侮辱、诈骗、违规导流 → 隐藏/举报/升级。

### 私信五步

```text
确认对方问题
→ 说明需要的最少信息
→ 判断买方/卖方和时间阶段
→ 预约 20 分钟诊断
→ 记录来源内容 ID 和 next_action_date
```

不得在私信中索取与当前诊断无关的身份证、账户、精确资产和家庭隐私；敏感材料只在进入正式交易且有必要时通过受控渠道收集。

## PC-20. 内容风险分级与应急

| 级别 | 示例 | 发布要求 |
|---|---|---|
| L1 低 | 通用买房框架、公开城市生活 | 编辑自检 |
| L2 中 | 项目优缺点、价格、成交、市场判断 | 主顾问 + 数据复核 |
| L3 高 | 客户案例、业主房源、纠纷、定制交付、纪录/冠军 | 授权 + 品牌/合规审稿 |
| L4 禁止 | 未授权隐私、伪造数据、收益承诺、虚假竞买、规避监管 | 不发布并上报 |

出现错误时：**先停止传播 → 保存原始记录 → 核验 → 在原平台清楚更正 → 通知受影响客户/业主 → 复盘规则**。不通过删帖掩盖重大错误。

## PC-21. AI 生成内容规则

国家有关办法已对 AI 生成合成的文本、图片、音频、视频和虚拟场景提出显式/隐式标识要求，并自 2025 年 9 月 1 日起施行。[^compliance-ai]

锦城豪宅内部规则：

- AI 可辅助选题、提纲、字幕、翻译、图表和初稿，但项目事实、价格、案例、合同和客户结论必须由人核验。
- 使用 AI 生成/合成的逼真人物、声音、房屋效果或场景时，按法律和平台要求进行标识，不让用户误认为真实交付或真实客户。
- 不用 AI 伪造客户评价、成交截图、业主授权、项目效果或马老师本人从未说过的话。
- 保存提示词、生成版本、人工修改和最终发布记录；平台已提供声明入口时必须使用。

## PC-22. 平台 KPI：从流量到交易结果

| 层级 | 指标 |
|---|---|
| 内容质量 | 留存、完播/阅读完成、收藏、分享、搜索进入、高质量评论 |
| 线索质量 | 有效私信、买/卖身份、预算/房源匹配、A/B 级占比 |
| 销售推进 | 诊断率、方案率、带看率、委托率、明确意向率 |
| 商业结果 | 内容辅助成交、业主直委、非公开房源、转介绍、复购 |
| 品牌资产 | “马老师/马国良/锦城豪宅”搜索一致性、核心栏目连续性 |
| 风险 | 错误更正、过期房源、隐私、侵权、违规营销和账号处罚 |

不得用单条爆款掩盖低质量线索和低成交能力。
