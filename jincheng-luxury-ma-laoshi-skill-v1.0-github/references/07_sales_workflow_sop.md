# 第九部分｜完整销售工作流

## 27. Core Workflow

```text
内容/转介绍/渠道线索
→ 线索识别
→ 高净值需求诊断
→ 客户决策图
→ 城市与板块假设
→ 一二手统一选筹
→ 房源核验与提纯
→ 三套以内方案
→ 带看设计
→ 带看复盘
→ 异议与家庭共识
→ 风险关闭
→ 谈判与交易结构
→ 签约及复杂交易管理
→ 交付与后服务
→ 案例沉淀
→ 复购/转介绍/内容反哺
```

## 28. 阶段 SOP

### S1 线索进入

必须记录：来源、首次需求、预算可信度、时间、决策人、下一步时间。

输出：`Lead Card`。

### S2 需求诊断

不急于推房，至少确认：

- 为什么买。
- 谁住、多久住一次。
- 上一套房哪里不满意。
- 最怕什么风险。
- 看过什么，为什么没买。
- 什么条件出现会立即行动。
- 购房是否依赖旧房出售、融资、企业回款或其他不确定资金。
- 若市场、收入或交付不如预期，客户能否承受并愿意继续持有。

输出：`Client Memo v1` + `Cycle Readiness`。

### S3 资格与合作判断

判断五项：

1. 需求真实性。
2. 资金可行性与家庭/企业安全边界。
3. 决策链可识别性。
4. 时间窗口与等待成本。
5. 客户是否认可专业价值和真实沟通。

不是所有高预算线索都进入高投入服务。

### S4 城市与板块论证

先判断城市和板块，再判断项目和房源。  
跨城市客户必须回答：“为什么不在另外一个城市买？”

输出：`Asset Thesis`。

### S5 房源资源提纯

把所有可能房源分为：

- 核心候选。
- 可谈判备选。
- 用于对比的排除项。
- 信息不完整的待核验项。

弱周期下额外记录：挂牌时长、历史调价、业主动机、同类真实成交、有效买方深度和可谈条款。

输出：`Property Source Sheet`。

### S6 方案与带看

每次不超过 3 套核心房源。  
带看顺序围绕验证假设，而不是按距离随意排列。

输出：`Viewing Plan`。

### S7 带看复盘

带看结束后记录：

- 第一反应。
- 最喜欢的三个点。
- 最不能接受的三个点。
- 与上一套相比的变化。
- 家庭成员意见。
- 真实价格承受和下一步。

24 小时内完成 `Viewing Debrief`。

### S8 共识与异议处理

把“再看看”拆成具体原因：

- 没有合适房源。
- 价格不确定。
- 家人未统一。
- 对交付不放心。
- 付款时间不匹配。
- 被其他中介或返佣影响。
- 客户其实尚未形成购买必要性。

每个异议都要有证据和关闭动作。

### S9 谈判

先完成内部底线和替代方案，再与对方谈。  
价格只是条件之一，还要处理付款、交付、装修、车位、家具、税费、腾退和时间。

输出：`Negotiation Plan`。

### S10 交易作战室

进入意向或合同阶段后，建立节点表，明确责任人和截止日期。

输出：`Deal War Room`。

### S11 交付与后服务

包含：验房、交付资料、装修设计、入住、物业关系、资产档案、后续出租出售与年度回访。

### S12 案例沉淀

每单无论成败，转成：

- 案例卡。
- 风险卡。
- 话术卡。
- 房源知识。
- 内容选题。
- 团队训练题。
