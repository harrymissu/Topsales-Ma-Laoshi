# 第二十部分A｜中国社交媒体公开情报审计

## SM-1. 研究口径与证据等级

### SM-1.1 证据等级

| 等级 | 定义 | 可用于什么 | 不可用于什么 |
|---|---|---|---|
| A | 官网、工商、平台认证主页、可打开的原始内容 | 品牌身份、公开业务、原始内容主题 | 推断实际成交和客户质量 |
| B | 平台公开搜索索引、账号公开视频标题与简介 | 判断公开内容方向、栏目和关键词 | 代表账号全部内容或后台表现 |
| C | 媒体采访、本人公开自述、第三方案例文章 | 提炼方法和行业经验 | 直接当作独立核验业绩 |
| D | 基于多条公开样本的专业推断 | 形成待验证假设 | 对外作为确定事实 |

### SM-1.2 研究输出的强制标签

每条社交情报必须标记为以下之一：

- `verified_public_fact`：可由官方页面或原始内容直接核验。
- `platform_visible_sample`：平台公开搜索可见样本，不代表全量。
- `self_reported`：账号简介、采访或本人公开自述。
- `expert_inference`：根据样本形成的专业推断，必须写清依据。
- `pending_backend_check`：需要抖音、微信、小红书或 CRM 后台核验。

### SM-2.1 公开内容资产的结构判断

锦城豪宅公众号已经形成三种能力：[^social-jc-media]

1. **市场解释能力**：年度报告、市场大事件、城市豪宅图鉴等内容建立宏观权威。
2. **项目教育能力**：板块、项目、户型和交付的长文有利于客户见面前自我教育。
3. **房源分发能力**：甄选房源数量较多，说明内容已承担库存曝光职能。

仍需补强的第四种能力是：

> **案例证据能力**——公开讲清客户遇到什么决策障碍、马老师做了什么、如何关闭风险，以及哪些结果不能复制或不能公开。

### SM-3.1 马老师的公开搜索问题

当前公开身份至少存在两个名称：

- 官网：**马老师 / 马国良**。
- 社交平台公开样本：**马老师｜锦城豪宅**。

这会造成三个问题：

1. “马老师”是高频通用称呼，搜索结果容易混入书法、培训、教育等无关内容。
2. 客户从短视频搜索实名时，可能无法快速确认与官网联合创始人是同一人。
3. 媒体引用、榜单统计和客户转介绍难以积累为同一个数字身份。

### SM-3.2 建议统一命名

主账号显示名优先使用：

> **马老师｜锦城豪宅**

简介首句绑定实名：

> 锦城豪宅联合创始人马国良，大家叫我马老师。

所有平台简介、公众号署名、视频口播、封面角标和搜索词统一包含：

```text
马老师｜马国良｜锦城豪宅｜成都豪宅｜麓湖｜金融城｜天西
```

“马老师”负责记忆，“马国良”负责实名核验，“锦城豪宅”负责品牌承接。项目词不堆在账号名中，放在标题、话题和合集。

## SM-4. 当前社交媒体优势

### SM-4.1 组织化内容能力

官网公开的团队结构显示，锦城豪宅已形成 TONY、马老师、小春哥、玥姐等多个团队。这个组织基础适合建立“品牌主账号 + 专家账号 + 板块账号”的矩阵，而不是让每个人重复拍同一套房。[^social-jc-about]

### SM-4.2 长短内容兼备

公众号约 170 篇长内容提供深度和搜索资产；抖音、哔哩哔哩等公开视频提供视觉入口。理想路径是：

```text
短视频激发问题
→ 长文完成教育
→ 私域完成诊断
→ 小程序/CRM 管理房源与进度
→ 案例内容反哺公域
```

### SM-4.3 产品和板块覆盖较宽

官网业务页公开列出成都天府新区、锦江区、武侯区以及麓湖、金融城等一二手豪宅项目，说明品牌具备构建城市级项目数据库的基础。[^social-jc-business]

### SM-4.4 品牌价值观与马老师方法一致

官网强调“客户立场、长期主义、精细化服务”；马老师补充信息强调坦诚、叫停客户、解决复杂问题。两者可以合并成更清晰的个人品牌承诺：

> **客户立场不是一句口号，而是敢劝停、敢讲缺点、敢对复杂问题负责。**

## SM-5. 当前公开内容缺口

### SM-5.1 个人能力证据不够集中

公开内容容易让人看到“马老师看过哪些豪宅”，但不一定能在 30 秒内回答：

- 他最擅长什么客户？
- 他解决过哪些别人解决不了的问题？
- 为什么客户不因返佣离开？
- 为什么业主愿意把稀缺房源交给他？

解决方式：将 3 个成功案例和 3 个失败案例做成长期置顶的“能力证据库”。

### SM-5.2 项目词强、客户问题词弱

“项目名 + 面积 + 价格 + 豪装”有搜索价值，但客户真正行动往往由以下问题驱动：

- 现在该不该置换？
- 新房还是二手？
- 3000 万应该买整层、别墅还是大平层？
- 家庭成员意见不一致怎么办？
- 资产好但交付不确定，是否值得等？

标题和选题应从“房是什么”升级为“客户为什么决策”。

### SM-5.3 “好房、天花板、捡漏”类词语需要证据

这类词能制造点击，但会削弱长期可信度。每次使用必须回答：

1. 比较范围是什么？
2. 评价维度是什么？
3. 数据和房源状态截至哪一天？
4. 哪类客户不适合？

证据不足时，改为“同类样本中的高位产品”“当前挂牌中相对有优势的个案”。

### SM-5.4 缺少业主端公开信任内容

豪宅业务的护城河不只是买方咨询，还包括：

- 业主直委。
- 非公开房源。
- 钥匙和看房权。
- 真实底价与出售动机。
- 对房源过度曝光的控制。

马老师需要建立“为什么业主愿意把房交给我”的内容支柱。

### SM-5.5 矩阵账号可能重复、线索难归因

团队账号多时，常见风险是：

- 同一房源多人重复发布，损伤稀缺感。
- 文案、价格和房源状态不一致。
- 客户被多人同时触达。
- 成交后无法判断哪条内容真正起作用。

必须建立统一内容 ID、房源 ID 和线索来源 ID。

### SM-6.1 马老师账号首页三条置顶

1. **我是谁**：12 年地产、开发商体系和豪宅成交数据，明确统计口径与证据状态。
2. **我解决过什么**：三个案例各用 20—40 秒讲“问题—动作—结果”。
3. **我如何服务**：从诊断、选筹、谈判到交付的 8 步流程，并明确“不适合买时会叫停”。

### SM-6.2 推荐账号简介

> 马老师｜锦城豪宅。实名马国良，锦城豪宅联合创始人。12 年地产销售，近 3 年专注成都豪宅交易；主攻高新区、麓湖、天西，服务企业家和高净值家庭。敢讲缺点，也负责把复杂问题解决到底。数据与案例以置顶说明为准。

对外发布前，必须把 12 年、3 年、成交额和套数更新为有凭证的同期口径。
