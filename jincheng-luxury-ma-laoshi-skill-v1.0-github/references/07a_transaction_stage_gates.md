# 第九部分A｜豪宅交易全流程与阶段策略

## TR-0. 本模块的作用

豪宅交易不能只用“带看—谈价—签约”概括。千万级以上交易通常同时存在三条链路：

```text
客户决策线：为什么买 → 谁拍板 → 何时行动 → 家庭是否共识
资产房源线：城市/板块 → 产品 → 具体房源 → 权属/交付/价格证据
交易执行线：意向 → 合同 → 资金 → 贷款/解押 → 税费/登记 → 交付
```

任何一条链路没有闭合，都可能在最后一刻失单或产生交付风险。本模块要求每一阶段都具备：**目标、证据、动作、责任人、截止时间、风险、退出条件和下一步**。

> 马老师负责建立判断、协调资源和推进结果；涉及法律、税务、贷款、评估、登记、工程质量等专业结论时，必须由相应机构或专业人士确认。

## TR-1. 四层签约前审查与阶段门

### 四层审查

| 层级 | 必须回答的问题 | 未通过时的处理 |
|---|---|---|
| 人 | 买卖双方身份、授权、婚姻/共有、决策人是否清楚 | 暂停推进，补授权或让真正决策人进入 |
| 房 | 权属、抵押、查封、租赁、用途、交付、装修和重大瑕疵是否核验 | 列为红色风险；无法关闭则停止 |
| 钱 | 预算、首付款、贷款、旧房出售、企业回款、税费和资金路径是否可执行 | 输出 WAIT / REDIRECT，不以情绪催签 |
| 约 | 价格、付款、违约、交付、家具、车位、定制等是否写入可执行合同 | 不接受“口头答应以后补” |

### 阶段门规则

1. **没有上一阶段输出，不进入下一阶段。**
2. **没有证据的优势只能写“待核验”，不能作为签约理由。**
3. **所有红色风险必须有责任人和关闭期限；不能关闭时必须明确 STOP。**
4. **每次客户沟通只推进一个最关键的不确定性，避免信息轰炸。**
5. **每个有效客户和每笔活跃交易必须有 `next_action_date`。**

## TR-2. 买方交易十九阶段总表

| 阶段 | 核心目标 | 马老师的策略与建议 | 标准输出 / 进入下一阶段条件 |
|---|---|---|---|
| G0 线索与来源 | 判断是否值得投入 | 记录平台、转介绍人、原始问题；不因高预算直接判定高质量 | `Lead Card`；已知预算、用途、时间、决策人和下一步 |
| G1 深度诊断 | 找到真正购买原因和否决项 | 问“为什么现在”“谁住”“上一套哪里不满意”“不买会怎样”；画家庭决策图 | `Client Memo v1`；客户认可问题定义 |
| G2 资金与资格 | 证明交易在家庭/企业现金流上可执行 | 压测首付、贷款、旧房出售、企业回款、税费和交付延迟；弱周期宁可 WAIT | `Capital & Eligibility Check`；资金路径和备用方案清楚 |
| G3 城市/板块论证 | 先确定方向再看具体房 | 对跨城客户回答“为什么是成都、为什么不是其他城市”；把通勤、家庭与退出客群写清 | `Asset Thesis`；板块范围收敛 |
| G4 一二手统一寻源 | 建立完整选择权 | 新房、二手、业主直委、非公开房源同池比较；不得因佣金限制选项 | `Property Source Sheet`；核心池、备选池、排除池形成 |
| G5 房源核验 | 把“听说”变成证据 | 核验权属、价格有效期、抵押/租赁、交付标准、噪音、景观、物业、业主动机与授权 | `Property Verification`；重大事实可证明 |
| G6 三套方案 | 降低选择噪音 | 最多三套，分别代表最适配、最稀缺、最稳妥；每套必须写“为什么不买” | `Shortlist + Comparison`；客户确认带看目标 |
| G7 带看设计 | 用现场验证假设 | 顺序按“基准—挑战—决赛”设计；提前明确每套要验证的问题和参与家庭成员 | `Viewing Plan`；时间、权限、资料、隐私确认 |
| G8 带看与记录 | 观察真实反应而非只讲卖点 | 少讲参数，多看动线、光线、噪音、尺度、车行、人行、物业和家庭互动 | 现场记录；房源事实与客户反应完整 |
| G9 24小时复盘 | 把情绪转为决策条件 | 问最喜欢/最排斥/与上一套变化/价格真实承受；不得只问“感觉怎么样” | `Viewing Debrief`；第一问题和下一动作清楚 |
| G10 家庭共识与异议 | 关闭真正的不确定性 | 把“再看看”拆成房源、价格、资金、家人、交付、返佣或必要性问题；必要时单独与关键成员沟通 | `Consensus Map`；异议有证据和关闭动作 |
| G11 出价与谈判 | 用结构而不只用低价成交 | 先定理想值、上限、BATNA；价格与付款、交付、家具、车位、税费、时间一起谈 | `Negotiation Plan`；客户书面确认边界 |
| G12 签约前四审 | 防止“谈成了却不能安全签” | 完成人、房、钱、约四审；合同、补充协议、清单和特殊承诺逐项确认 | `Pre-contract Checklist` 全部通过或风险被书面接受 |
| G13 合同/网签/资金监管 | 把商业共识变成可执行文件 | 使用当期正规系统和机构；特殊承诺落纸；不让中介个人代收房款 | 已签文件、网签/备案状态、监管账户与付款节点明确 |
| G14 贷款/解押/带押安排 | 管理银行和时间依赖 | 提前预审贷款；二手抵押房判断解押或当前可用的“带押过户”路径；准备替代银行/时间表 | `Financing & Mortgage Map`；银行受理和条件确认 |
| G15 税费/登记/放款 | 完成产权与价款安全转移 | 税费只做信息协调，以税务和登记口径为准；盯住材料、预约、登记状态和放款条件 | 登记受理/完成、资金释放条件达成 |
| G16 交付与验收 | 确保客户拿到“谈判时承诺的房” | 新房查交付标准和整改；二手清点家具设备、钥匙、车位、物业费、租约和腾退 | `Handover Checklist` 签认；遗留问题有期限 |
| G17 后服务与资产档案 | 把一次成交变成长关系 | 设计、装修、物业、出租出售、年度估值和家庭变化回访；不把交付视为结束 | `Asset File` + 回访计划 + 转介绍触发点 |
| G18 案例与内容沉淀 | 把结果转为组织能力 | 内部保留完整事实，对外严格脱敏；失败同样形成规则 | `Case Card`、风险卡、内容 Brief、训练题 |

## TR-3. 前端阶段策略：G0—G6

### G0—G1：线索与诊断

**策略：**

- 公域线索先识别“问题质量”，不先判断客户身价。
- 转介绍线索仍需重新诊断，不能把介绍人的描述当客户真实需求。
- 第一次沟通的目标不是约看房，而是得到下一次有价值沟通的许可。
- 诊断结束必须复述：“我理解您真正要解决的是……，不是简单找一套更贵的房。”

**失败案例强制规则：**因未及时跟进而丢失大额客户的教训，落实为 `next_action_date` 必填；A 类客户在约定时间内必须有实质进展或主动说明。

### G2：资金与资格

**弱周期策略：**

- 不把“总资产高”自动等同于“当前流动资金充足”。
- 企业家客户至少询问：购房款是否影响经营安全垫、是否依赖应收账款或旧房出售、最坏情景能否持有三年以上。
- 若资金链依赖多个未确定事件，输出 WAIT 或 REDIRECT，而不是用“窗口马上关闭”催单。
- 贷款、税费、购房资格和跨境资金安排必须由当期专业渠道确认。

### G3：城市与板块

**策略：**

- 自住优先从日常时间表反推板块，不从项目名反推需求。
- 第二居所重点核验使用频率、维护成本、到达便利和空置安全。
- 资产属性重点核验未来接盘客群、供应替代和退出难度，不用“豪宅都抗跌”概括。
- 对四川及中西部客户同时比较成都落脚价值与原城市持有成本。

### G4—G5：寻源与核验

**策略：**

- 每个需求至少做一次新房/二手交叉核验，防止只看单一渠道。
- 房源分为 A 级直委/独家、B 级稳定合作、C 级待核验、D 级仅作市场线索。
- 二手房必须记录挂牌时长、历史调价、业主动机、看房反馈和同类替代。
- 新房必须记录可售状态、交付证据、特殊条款权限、价格有效期和开发商决策链。
- 非公开房源先确认传播边界，不因内容需求破坏业主信任。

**失败案例强制规则：**因只在原有房源池中寻找而被其他中介用二手房成交的教训，落实为“一二手统一选筹”和“未覆盖替代房源不得宣称已穷尽市场”。

### G6：三套方案

每套房必须回答六句话：

1. 它解决客户哪个第一问题。
2. 它相对另外两套牺牲了什么。
3. 价格证据来自哪里、截至何时。
4. 最大不可修复缺点是什么。
5. 什么客户不适合买。
6. 什么条件出现时应放弃。

## TR-4. 决策与谈判阶段策略：G7—G12

### G7—G9：带看、现场和复盘

**带看不是讲解会，而是实验。**

- 第一套建立基准，第二套挑战客户原有偏好，第三套用于决赛判断。
- 豪宅现场必须观察：门厅与归家、车库到室内、保姆/服务动线、主人区与家庭区、噪音、遮挡、采光、私密、收纳、设备、会所和物业响应。
- 不在业主家公开拍摄门牌、人脸、证件、艺术品细节、安防系统和可识别家庭信息。
- 复盘在 24 小时内完成；现场兴奋时不立即催定，先做冷静检查。

### G10：家庭共识

| 表面异议 | 可能真实问题 | 建议动作 |
|---|---|---|
| “再看看” | 选择标准不清 | 重新排序必须项和否决项 |
| “价格高” | 不相信价值或资金不舒服 | 分开做价格证据与现金流压力测试 |
| “家里不同意” | 决策人未进入或生活场景不一致 | 让关键成员参与一次专门沟通/带看 |
| “担心交付” | 缺证据而非缺承诺 | 查历史交付、合同标准、样板差异和整改机制 |
| “别人返得更多” | 顾问价值未量化 | 展示房源、谈判、风险和交付价值；不攻击同行 |

### G11：谈判

```text
先谈目标与替代方案
→ 再核验对方动机与决策人
→ 决定出价锚点和节奏
→ 每一次让步换回一个条件
→ 所有特殊承诺进入书面文件
```

**弱周期建议：**

- 买方不要把“市场弱”理解为所有卖方都必须接受低价；先判断具体业主动机和替代买方。
- 卖方不要用历史高点作为唯一锚；要比较真实成交、当前有效买方和持有时间成本。
- 价格不是唯一变量，可交换付款速度、交付时间、家具设备、车位、租约、装修、定制和保密。

**反佣策略：**明确收费、服务清单、关键动作、潜在损失避免和交付责任；客户若只选择极端反佣，应尊重其选择，但不做无法兑现的承诺。

### G12：签约前四审

签约前必须召开一次内部 15—30 分钟“红线会”，逐项确认：

- 身份、授权、共有和决策权。
- 权属、抵押、查封、租赁、用途与重大瑕疵。
- 付款来源、贷款、旧房出售、税费和资金监管路径。
- 合同主体、价格、付款、违约、交付、清单和特殊承诺。

任何人说“这个通常没问题”都不能替代书面证据。

## TR-5. 执行与交付阶段策略：G13—G18

### G13：合同、网签/备案与资金监管

- 成都现行住房交易已提供“蓉易住”等线上办理路径，但入口、适用条件和办理步骤会更新，必须在签约当天重新核验。[^transaction-rongyizhu]
- 经纪促成的二手交易应按当期监管要求使用正规经纪和网签路径；不要把经纪促成交易伪装为“自由成交”。
- 房款不得进入经纪个人账户；资金监管、付款释放条件和收款主体必须写清。
- 合同、补充协议、附件、家具设备清单、装修/定制标准必须相互一致。

### G14：贷款、解押与“带押过户”

- 贷款预审应早于正式出价，避免签约后才发现额度、期限或主体不符合。
- 抵押房须建立“原贷款—解押/带押—新贷款—监管—登记—放款”时间图。
- 成都已公开推进存量房贷款资金监管和跨行“带押过户”，但参与银行、业务条件和流程必须以当期银行及官方系统为准。[^transaction-fundsupervision]
- 任何银行口头意见都应转为受理材料或书面确认；同时准备备用银行和延期条款。

### G15：税费、登记与放款

- 不在 Skill 中固定税率或资格结论；根据房屋、家庭、持有、主体和当期政策由税务/登记/贷款机构确认。
- 交易作战室每日更新：材料状态、预约、受理、补件、登记、抵押、放款和监管解付。
- 重大节点完成后向客户发“一页状态更新”，避免信息焦虑。

### G16：交付与验收

**新房：**交付文件、面积/户型、装修标准、设备、渗漏空鼓、门窗、智能系统、车位、物业、整改期限和产权办理资料。

**二手：**腾退、钥匙、门禁、家具艺术品/设备清单、物业水电气、停车、会籍、租约、维修记录、房屋现状和尾款释放条件。

豪宅交付建议使用照片、视频、清单和双方签字留档；涉及隐私的资料只保存在受控空间。

### G17—G18：后服务、案例与转介绍

- 交付后 7 天、30 天、半年和年度节点提供不同服务，而不是当天索要转介绍。
- 先解决真实问题，再在客户明确满意的时点提出：“身边有人遇到类似复杂问题，可以把我介绍给他。”
- 对外案例只保留问题、方法和边界；客户身份、精确行踪、资金、房号、聊天和家庭关系默认删除。

## TR-6. 新房豪宅交易专线

> 一手房是时机的生意：机会来自供应节奏、货值结构、开发商授权、交付能力和条款窗口，而不是盲目抢购。

| 节点 | 必查事项 | 马老师策略 | 停止/升级条件 |
|---|---|---|---|
| 项目准入 | 开发主体、销售许可/现售状态、土地和规划、交付历史 | 先做开发商与项目风险底稿，再谈产品 | 主体、许可或重大规划信息不清 |
| 供应与选房 | 楼栋、楼层、朝向、噪音、遮挡、货值和保留房源 | 识别真实稀缺与营销稀缺；建立同类替代 | 只靠口头说“最后一套” |
| 样板与交付 | 样板间、非交付展示、材质品牌、设备型号、变更权 | 对比合同附件和历史交付，不用样板情绪替代证据 | 关键标准不能写入合同 |
| 认购/定金 | 主体、房号、金额、退转条件、签约期限 | 先审重要条款再付款；性质与后果让法务确认 | 要求先付后看合同或收款主体异常 |
| 价格与优惠 | 总价、付款、折扣、佣金、车位、物业、定制 | 把所有条件换算成总成本和确定性 | 优惠依赖不可证明的未来承诺 |
| 合同与备案 | 主合同、补充协议、附件、电子签约/备案 | 特殊条款逐项编号；保留完整版本和签署证据 | 主附件冲突或口头承诺拒绝落纸 |
| 工程跟进 | 进度、设计变更、材料替换、停复工和交付通知 | 关键节点图像留档；异常及时升级开发商 | 重大延期/变更无正式方案 |
| 交付与整改 | 文件、验房、装修、设备、物业、整改期限 | 专业验房 + 交付清单 + 问题闭环 | 重大质量/安全问题未形成处理方案 |
| 产权与后服务 | 登记资料、车位、物业、装修入住 | 建立资产档案和年度项目更新 | 权利边界或资料长期缺失 |

**半定制/定制交付特别规则：**设计图、材料品牌型号、施工范围、变更机制、验收标准、工期、违约、保修和第三方责任必须书面化。MG-001 的方法不是“开发商可以通融”，而是把不匹配的供给重构为可验收的交付方案。

## TR-7. 二手豪宅交易专线

> 二手房是时间的生意：价值来自业主关系、信息持续更新、买卖双方触发事件和复杂后端执行。

| 节点 | 必查事项 | 马老师策略 | 停止/升级条件 |
|---|---|---|---|
| 委托与授权 | 产权人、共有人、婚姻、代理、授权范围、出售意愿 | 与真正有处分权的人沟通；核验发布边界 | 委托主体和产权主体不一致 |
| 权属与限制 | 不动产权、抵押、查封、租赁、居住/使用安排 | 建立权属风险图，必要时法务/登记核验 | 查封、权利争议或无法解释的限制 |
| 房屋现状 | 装修、设备、渗漏、改造、违建可能、噪音和物业 | 现场、资料、物业和历史维修多源核验 | 重大瑕疵被隐瞒或无法界定 |
| 价格与动机 | 挂牌史、调价史、持有成本、出售原因和时间压力 | 用成交、替代、买方池和时间成本定价 | 业主只接受脱离市场的历史锚点且拒绝试验 |
| 买方筛选 | 预算、资金、身份、时间、隐私与真实需求 | 先筛选再带看；高隐私房源分级开放 | 买方身份/资金不明却要求完整资料 |
| 带看与反馈 | 看房权限、人数、拍摄、安保、反馈 | 统一反馈格式，保护业主家和邻里隐私 | 偷拍、直播、泄露门牌或安防细节 |
| 意向与谈判 | 出价、定金/意向安排、付款、家具、腾退 | 价格和非价格条件一起谈；记录每次让步 | 以虚假竞买抬价或关键条款模糊 |
| 合同与监管 | 网签、资金监管、贷款、解押/带押、税费、交付 | 交易作战室管理多方节点；房款不经个人 | 资金路径不安全或合同主体异常 |
| 登记与放款 | 材料、税费、转移登记、抵押、监管解付 | 每一节点发送状态更新和补件清单 | 关键条件未满足却要求提前放款/交房 |
| 腾退与交付 | 家具、艺术品、设备、钥匙、物业、租约、维修 | 用清单、影像、表计和签字完成交接 | 房屋状态与约定明显不一致 |

MG-003 的方法应固化为：**稀缺房源获取 + 权属/资金/后端问题作战室 + 结果留档**，而不是只宣传“打破纪录”。

## TR-8. 卖方业主十一阶段流程

| 阶段 | 业主需要的结果 | 马老师的建议 |
|---|---|---|
| O1 出售诊断 | 明确为什么卖、何时需要钱、是否置换 | 先判断卖不卖、现在卖还是等，不默认接委托 |
| O2 处分权审查 | 确认产权人、共有人和授权 | 所有关键权利人尽早进入，避免成交后否决 |
| O3 资产体检 | 形成产品、瑕疵、权属、装修、物业和竞品底稿 | 先找真实缺点再包装，不隐瞒重大事项 |
| O4 定价三角 | 市场成交 × 在售替代 × 业主时间成本 | 给出快速兑现、平衡出售、私密试探三种方案 |
| O5 隐私分级 | 明确公开、受控、私密信息 | 门牌、户型细节、藏品、安防、家庭行踪分级 |
| O6 内容与资料 | 形成真实、克制、可验证的房源资产包 | 公开版不等于完整资料；高敏资料只给合格买方 |
| O7 渠道与买方池 | 选择主账号、合作经纪和目标客户 | 不盲目全网铺货；避免价格、状态和文案冲突 |
| O8 带看与反馈 | 得到真实买方反应 | 每次带看后反馈：喜欢、不喜欢、价格、下一步 |
| O9 周期与调价 | 判断继续、调整或暂停 | 用有效买方和替代房源变化，不用经纪人主观催降 |
| O10 谈判与交易 | 控制价格、付款、隐私和交付 | 只与有能力执行的买方进入深谈 |
| O11 交付与复盘 | 安全收款、腾退、交接和后续资产安排 | 周报直到资金和交付完全闭环 |

### 业主周报固定结构

```text
本周曝光范围 → 合格买方数 → 带看与反馈 → 同类成交/调价
→ 价格判断是否变化 → 风险与下一步 → 需要业主决策的事项
```

## TR-10. 红黄绿风险与停止条件

### 红色：立即停止或升级专业机构

- 身份、产权、授权、查封或重大权利存在实质疑点。
- 房款要求进入个人或非约定主体账户。
- 对方要求隐瞒真实成交价、虚构交易、伪造材料或规避监管。
- 重大交付、质量或产权承诺拒绝写入合同。
- 客户资金依赖未确认融资且无备用方案。
- 客户/业主要求泄露他人隐私或偷拍。

### 黄色：设期限关闭

- 抵押、租赁、共有人意见、贷款预审、税费口径、家具清单、腾退时间未确认。
- 房源价格或状态超过约定核验时效。
- 家庭关键决策人未参与。
- 开发商/业主的特殊优惠仍为口头表达。

### 绿色：可进入下一阶段

- 关键事实有来源，责任人清楚，客户理解风险，合同与资金路径可执行。

## TR-11. 弱经济环境下的流程化应对

| 流程阶段 | 弱周期客户心理 | 经纪人应做 | 经纪人不应做 |
|---|---|---|---|
| 诊断 | 担心收入、企业现金流和房价继续调整 | 做下行情景、持有能力与替代方案 | 用焦虑、政策或“抄底”催促 |
| 寻源 | 选择多但真假混杂 | 提纯房源、核验动机、扩大一二手覆盖 | 把挂牌多等同于都能低价买 |
| 带看 | 容易反复比较 | 建立决策标准和停止看房条件 | 无限带看、堆项目 |
| 谈判 | 希望低价又怕错过 | 识别具体对方压力和可交换条件 | 虚构竞买或普遍性“底价” |
| 签约 | 更关注安全和退出 | 强化资金、合同、交付和备用路径 | 为了冲业绩跳过审查 |
| 交付 | 对延迟和质量更敏感 | 提前介入、留证、整改闭环 | 签约后降低服务强度 |
| 后服务 | 更重视资产运营 | 提供真实估值、出租出售、物业与置换服务 | 只在客户再次买房时联系 |

## TR-14. 交易信息时效与法律边界

- 地方系统、资格、税费、贷款、资金监管、带押过户和登记流程会变化；本 Skill 提供的是管理框架，不是固定办事指南。
- 每笔交易在 G12 阶段生成一份“当日有效信息确认单”，记录来源、确认人和时间。
- AI 不得自行给出确定税额、法律效力、贷款批复或产权结论；必须标注“以相应专业机构和正式文件为准”。
- 交易材料遵循最小必要和分级权限；身份证、账户、资产、行踪、家庭关系等不得进入普通内容库。
