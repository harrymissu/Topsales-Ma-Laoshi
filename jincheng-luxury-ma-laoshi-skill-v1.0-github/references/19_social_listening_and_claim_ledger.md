## SL-2. 月度社交情报报告

每月只回答十个问题：

1. 哪三个客户问题增长最快？
2. 哪三个项目被讨论最多？
3. 哪些讨论是事实，哪些是情绪？
4. 马老师的哪类内容带来最多有效咨询？
5. 哪类内容只带来围观？
6. 业主因什么内容主动联系？
7. 哪个同行方法值得试验？
8. 哪个公开观点需要更正或补证据？
9. 多账号是否出现重复、冲突或撞客？
10. 下月停止、继续和新增什么？

## SL-4. 内容审稿流程

```text
选题
→ 事实与房源状态核验
→ 客户/业主隐私检查
→ 法律和广告表达检查
→ 项目与同行表述检查
→ 发布
→ 24小时评论与错误监测
→ 30天业务转化复盘
→ 更新或下架过期内容
```

## SL-5. 多账号治理

### 发布前

- 房源负责人确认价格、状态和公开等级。
- 内容运营查询是否已有同项目、同户型、同角度内容。
- 同一非公开房源只允许一个主发布账号。
- 其他账号只能转发或使用差异化角度，并保留房源 ID。

### 线索进入后

- 以首次有效来源创建 `lead_id`。
- 同一客户只设一个主责任人。
- 其他账号的后续触达由主责任人协调。
- 内部协作佣金和归属按公司制度，不在客户面前争抢。

### 房源变化后

- 调价、成交、撤牌和暂停必须同步更新所有内容。
- 已失效内容加醒目标记或下架，不能继续以原状态获客。

## SL-6. 舆情和错误响应

出现事实错误、客户投诉或隐私风险时：

1. 先暂停传播和投放。
2. 保存原内容、评论、时间和责任链。
3. 24 小时内核验事实；涉及隐私应立即处理，不等完整调查。
4. 错误属实时，公开更正，不静默修改造成二次误导。
5. 对客户或业主直接道歉并说明补救措施。
6. 将事件转化为内部防错规则，不拿客户事件做流量内容。
